Homepage

In this application, I have set up the following scenarios using CanActivate and CanDeactivate:

  • When attempting to navigate to the Contact Us page, users must be logged in (with valid credentials) to access it.
  • If the user has not logged in and tries to access the page, they are redirected to the login page.
  • If the user is in the middle of filling out the contact form and tries to navigate away, the CanDeactivate guard will prevent them from leaving without confirmation, ensuring no data is lost.
